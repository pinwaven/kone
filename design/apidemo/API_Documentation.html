1 WLK 设备命令 API 文档

1.1 概述

  • 通过 UART 串口通信协议接收和处理命令
    • BAUDRATE: 230400
    • WordLen: 8bits
    • StopBits: 1bit
    • Parity: None
  • 采用基于帧的通信机制

1.2 通信帧结构

1.2.1 命令帧格式

[ADDR] [CMD] [DATA...] [CRC]
  |      |      |       |
  |      |      |       +-- CRC校验字节
  |      |      +---------- 命令数据 (可变长度)
  |      +----------------- 命令字节
  +------------------------ 设备地址字节 (固定为 0x21 '!')

1.2.2 响应帧格式

1.2.2.1 正常响应帧

[ADDR] [|] [RESPONSE_DATA...] [CRC]
  |     |         |            |
  |     |         |            +-- CRC校验字节
  |     |         +--------------- 响应数据内容
  |     +--------------------- 分隔符 '|'
  +--------------------------- 设备地址字节 (固定为 0x21 '!')

1.2.2.2 错误响应帧(主要是CRC错误、底层命令参数错误)

[ADDR] [?|] [ERROR_MESSAGE...] [CRC]
  |     |         |              |
  |     |         |              +-- CRC校验字节
  |     |         +----------------- 错误信息内容
  |     +----------------------- 错误标识 '?|'
  +----------------------------- 设备地址字节 (固定为 0x21 '!')

1.3 系统信息

  • 当前固件版本: v0.1.1

  • 设备地址: 0x21 (‘!’)

  • 通信协议: UART + 基于帧的协议 + CRC校验

  • 帧最大长度: 64字节 (MCU <– PC)

1.6 命令列表

1.6.1 帧结构说明

所有命令都遵循统一的帧结构。下面列出各命令的具体帧格式：

1.6.1.1 简单命令帧 (无参数)

[0x21] [CMD] [CRC]

1.6.1.2 复杂命令帧 (带参数)

[0x21] [CMD] [PARAM1] [PARAM2] ... [CRC]

1.6.2 1. 基础命令

1.6.2.1 1.1 CMD_POLL (0x01) - 系统状态轮询

  • 命令帧: [0x21] [0x01] [CRC]

  • 用途: 获取系统当前状态信息

  • 参数: 无

  • 响应格式:

    !|a:[action_type],s:[status]-[error_str],m:[h_motor_state]-[v_motor_state]`

  • 响应示例:

    !|a:0,s:IDLE-null,m:0-0
!|a:1,s:HOMING_V_MOTOR-moo,m:0-1
!|a:4,s:SCAN:1234-len:1234,m:1-0

说明:

  • a: 当前动作类型 (0=NONE, 1=HOMING, 2=MOVE_DURATION, 3=MOVE_TO_SS, 4=SCAN, 6=READ_QR)
  • s: 状态字符串-错误信息
  • m: 水平电机状态-垂直电机状态 (0=IDLE, 1=MOVING, 2=STOP, 3=ERROR)

1.6.2.2 1.2 CMD_HI (0x02) - 获取设备信息

  • 命令帧: [0x21] [0x02] [CRC]

  • 用途: 获取固件版本和唯一ID

  • 参数: 无

  • 响应格式: !|ver:[version]~[uid_hex]

  • 响应示例:

    !|ver:v0.1.1~1234567890abcdef12345678

1.6.2.3 1.3 CMD_TEST (0x03) - 测试命令(不要使用)

  • 命令帧: [0x21] [0x03] [CRC]
  • 用途: 保留
  • 参数: 无
  • 响应格式: !|test:[val1], [val2], [val3]

1.6.3 2. GPIO 控制命令

1.6.3.1 2.1 CMD_GPIO_READ (0x06) - 读取GPIO状态

  • 用途: 读取所有GPIO引脚状态和ADC值

  • 参数: 无

  • 响应格式: !|ss_p:[val],ss_ud:[val],ss_start:[val],ss_stop:[val],ss_card:* [val],ld_p:[val],qr_tgl:[val],qr_rst:[val],adc:[adc1]-[adc2]

  • 引脚说明:

    • ss_p: 传感器电源
    • ss_ud: 传感器上下方向
    • ss_start: 起始传感器
    • ss_stop: 停止传感器
    • ss_card: 卡片传感器
    • ld_p: 激光器电源
    • qr_tgl: QR扫描器触发
    • qr_rst: QR扫描器复位

1.6.3.2 2.2 CMD_GPIO_WRITE (0x07) - 写入GPIO状态 (谨慎使用)

  • 用途: 控制特定GPIO引脚输出
  • 格式: ![0x07][pin_id][pin_val][CRC]
  • 参数:
    • pin_id: 引脚ID (0-7)
    • pin_val: 引脚值 (0/1)

响应格式: !|gpio_write:[pin_id],[pin_val]

1.6.4 3. 寄存器操作命令

1.6.4.1 3.1 CMD_REG_WRITE (0x04) - 写寄存器

  • 用途: 向TMC2300电机驱动器写入寄存器
  • 格式: ![0x04][node_addr][reg][data_4bytes][CRC]
  • 参数:
    • node_addr: 节点地址
    • reg: 寄存器地址
    • data_4bytes: 4字节数据
  • 响应格式: !|reg_write:[node_addr],[reg],0x[value]

1.6.4.2 3.2 CMD_REG_READ (0x05) - 读寄存器

  • 用途: 从TMC2300电机驱动器读取寄存器
  • 格式: ![0x05][node_addr][reg][CRC]
  • 参数**
    • node_addr: 节点地址
    • reg: 寄存器地址
  • 响应格式:
    • 成功: !|reg_read:[node_addr],[reg],0x[value]
    • 失败: !|reg_read:[node_addr],[reg],err:[error_code]

1.6.5 4. 动作命令 (需要轮询状态)

1.6.5.1 4.1 CMD_ACTION_HOMING (0x40) - 归零操作

  • 用途: 执行垂直和水平电机的归零操作
  • 参数: 无
  • 响应格式: !|homing:[result]
  • 轮询状态 (通过CMD_POLL获取):
    • HOMING_START: 开始归零
    • HOMING_V_MOTOR: 垂直电机归零中
    • HOMING_V_COMPLETE: 垂直电机归零完成
    • HOMING_H_MOTOR: 水平电机归零中
    • HOMING_H_COMPLETE: 水平电机归零完成
    • HOMING_COMPLETE: 归零完成
  • 结果码:
    • 0: A_REV_DOING - 正在执行
    • -1: A_REV_BUSY - 系统忙
    • -2: A_REV_BAD_PARAM - 参数错误

1.6.5.2 4.2 CMD_ACTION_MOVE_DURATION (0x41) - 定时运动

  • 命令帧: [0x21] [0x41] [motor_id] [velocity_4bytes] [duration_4bytes] [CRC]
  • 用途: 控制指定电机以特定速度运动指定时间
  • 帧长度: 12字节
  • 参数:
    • motor_id: 电机ID (0=水平, 1=垂直)
    • velocity: 目标速度 (int32, 单位: 步/秒)
    • duration: 运动时间 (uint32, 单位: 毫秒)
  • 响应格式: !|move_dur:[result].[motor_id],[velocity],[duration]
  • 轮询状态: MOVING

1.6.5.3 4.3 CMD_ACTION_MOVE_TO_SS (0x42) - 移动到传感器

  • 命令帧: [0x21] [0x42] [motor_id] [velocity_4bytes] [duration_4bytes] [ss_id] [CRC]
  • 用途: 控制电机移动直到触发指定传感器
  • 帧长度: 13字节
  • 参数:
    • motor_id: 电机ID (0=水平, 1=垂直)
    • velocity: 目标速度 (int32)
    • duration: 最大运动时间 (uint32, 毫秒)
    • ss_id: 传感器ID (0/1)
  • 响应格式: !|move_to_ss:[result],[motor_id],[velocity],[duration],[ss_id]
  • 轮询状态: MOVING

1.6.5.4 4.4 CMD_ACTION_SCAN (0x44) - 激光扫描

  • 命令帧: [0x21] [0x44] [velocity_4bytes] [duration_4bytes] [CRC]
  • 用途: 执行激光扫描，同时移动和采集ADC数据
  • 帧长度: 11字节
  • 参数:
    • velocity: 扫描速度 (int32, 步/秒)
    • duration: 扫描时间 (uint32, 毫秒, 最大60秒)
  • 响应格式: !|scan:[result],[velocity],[duration]
  • 轮询状态:
    • SCAN:[count]: 扫描中，显示当前采集的数据点数量
    • 完成后错误信息显示: len:[total_count]
  • 结果码:
    • -3: A_REV_NEED_HOMING - 需要先归零

1.6.5.5 4.5 CMD_ACTION_READ_QR (0x46) - QR码扫描

  • 用途: 启动QR码扫描操作
  • 参数: 无
  • 响应格式: !|read_qr:[result]
  • 轮询状态:
    • QR_START: 开始扫描
    • QR_TRIGGER_LOW: 触发信号拉低
    • QR_SCANNING: 扫描中
    • QR_DATA_RECEIVED: 数据接收完成
    • QR_COMPLETE: 扫描完成
    • QR_TIMEOUT: 超时 (3秒)
    • QR_ERROR: 错误
  • 成功时错误信息显示: QR:[扫描到的数据]

1.6.5.6 4.6 CMD_ACTION_CANCEL (0x08) - 取消当前动作

  • 用途: 取消当前正在执行的动作
  • 参数: 无
  • 响应格式: !|cancel:[result]
  • 说明: 会停止所有电机运动，关闭激光器，重置QR扫描器

1.6.6 5. 数据查询命令

1.6.6.1 5.1 CMD_ACTION_QUERY_DATA (0x45) - 查询扫描数据

  • 用途: 获取最近一次扫描的ADC数据
  • 参数: 无
  • 响应格式:
    • 无数据: !|no_data
    • 有数据: 返回包含所有ADC采样点的数据包
  • 数据包格式: 直接传输ADC缓冲区数据

1.7 动作状态机

1.7.1 动作状态说明 (action_state_t)

  • ACTION_STATE_IDLE: 空闲状态
  • ACTION_STATE_RUNNING: 运行中
  • ACTION_STATE_COMPLETED: 完成
  • ACTION_STATE_ERROR: 错误
  • ACTION_STATE_CANCELLED: 已取消

1.7.2 电机状态 (motor_state_t)

  • MOTOR_STATE_IDLE (0): 空闲

  • MOTOR_STATE_STOP (2): 停止
  • MOTOR_STATE_ERROR (3): 错误

1.8 通信协议注意事项

  1. CRC-8校验: 每个命令和响应都包含1字节CRC-8校验值，确保数据传输完整性
  2. 响应格式:
    • 正常响应: !|[响应内容]
    • 错误响应: !?|[错误信息] (底层错误、CRC错误、参数错误等)
  3. 超时处理: 动作命令有超时机制，超时后自动进入错误状态
  4. 状态轮询: 长时间动作需要通过CMD_POLL持续监控状态
  5. 互斥性: 同时只能执行一个动作命令
  6. 归零要求: 扫描功能需要先执行归零操作
  7. 帧长度: 所有通信帧不超过64字节

1.8.1 常见错误响应示例

!?|crc error                    # CRC校验失败
!?|bad cmd: (0xFF)             # 未知命令
!?|bad gpio_write: (0x07)      # GPIO写入参数错误
!?|my addr should: 0x21, but got: 0x22, len(5)  # 地址错误

1.9 错误代码

详细的动作命令返回值请参考上方的 动作命令返回值索引 部分。

  • A_REV_DOING (0): 正在执行
  • A_REV_BUSY (-1): 系统忙，请稍后重试
  • A_REV_BAD_PARAM (-2): 参数错误
  • A_REV_NEED_HOMING (-3): 需要先执行归零操作

1.10 使用示例

1.10.1 基本工作流程

  1. 发送 CMD_HI 确认设备连接
  2. 发送 CMD_ACTION_HOMING 执行归零
  3. 通过 CMD_POLL 监控归零状态直到完成
  4. 执行其他动作命令 (扫描、移动等)
  5. 通过 CMD_POLL 持续监控状态

1.10.2 扫描流程

  1. 确保已归零 (homed=1)
  2. 发送 CMD_ACTION_SCAN 启动扫描
  3. 通过 CMD_POLL 监控扫描进度
  4. 扫描完成后使用 CMD_ACTION_QUERY_DATA 获取数据

1.11.1 简单命令示例

1.11.1.1 发送POLL命令

1.11.1.3 错误响应示例

# CRC错误
发送: [0x21] [0x01] [0xFF]  # 错误的CRC
接收: [0x21] [0x3F] [0x7C] [crc error] [0xCRC]

# 未知命令
发送: [0x21] [0xFF] [0xCRC]  # 无效命令

# 地址错误

1.11.2 复杂命令示例

1.11.2.1 发送移动命令 (水平电机，速度1000步/秒，持续2秒)

发送: [0x21] [0x41] [0x00] [0xE8] [0x03] [0x00] [0x00] [0xD0] [0x07] [0x00] [0x00] [0xCRC]
      |---- |---- |---- |----------velocity---------|  |-------duration-------|  |CRC|
      ADDR  CMD   motor=0  1000 (0x03E8 → E8,03,00,00)  2000ms (0x07D0 → D0,07,00,00)
      
接收: [0x21] [0x7C] [move_dur:0.0,1000,2000] [0xCRC]

1.11.2.2 发送扫描命令 (速度500步/秒，持续5秒)

发送: [0x21] [0x44] [0xF4] [0x01] [0x00] [0x00] [0x88] [0x13] [0x00] [0x00] [0xCRC]
      |---- |---- |----------velocity---------|  |-------duration-------|  |CRC|
      ADDR  CMD   500 (0x01F4 → F4,01,00,00)    5000ms (0x1388 → 88,13,00,00)
      
接收: [0x21] [0x7C] [scan:0,500,5000] [0xCRC]

1.11.3 数据类型说明

  • int32: 4字节有符号整数，小端序
  • uint32: 4字节无符号整数，小端序
  • uint8: 1字节无符号整数
  • CRC: 1字节CRC-8校验值

1.11.4 CRC-8 算法详细说明

采用8位CRC校验算法，参数如下：

  • 多项式: x²+x¹+x⁰ = 0x07
  • 初始值: 0x00
  • 异或输出: 0x00
  • 输入反转: True (refin=True)
  • 输出反转: False (refout=False)

Python实现示例:

import anycrc
crc8 = anycrc.CRC(width=8, poly=0x07, init=0x00, xorout=0x00, refin=True, refout=False)
crc_value = crc8.calc(data_bytes)

CRC计算示例:

# 计算POLL命令的CRC
data = [0x21, 0x01]  # 地址 + 命令
crc8 = anycrc.CRC(width=8, poly=0x07, init=0x00, xorout=0x00, refin=True, refout=False)
crc_value = crc8.calc(data)
# 完整帧: [0x21, 0x01, crc_value]

1.11.5 字节序说明

所有多字节数据采用小端序 (Little Endian)格式：

  • 低位字节在前，高位字节在后
  • 例如：1000 (0x03E8) → [0xE8] [0x03] [0x00] [0x00]
